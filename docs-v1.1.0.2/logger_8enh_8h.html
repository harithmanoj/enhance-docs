How To Use

  • Include file logger.enh.h
  • Make sure to add logger.cpp to compilation
  • Use the macros O#_LOG_LINE, O#_LOG_DESC, O#_LOG_VAL where # in [1,5] in code at specific logging points.
    • O#_LOG_LINE where # in [1,5] takes no arguments and logs the completion of that line in that function.
    • O#_LOG_DESC where # in [1,5] takes one argument that is a string and logs the value of the string and current line and function.
    • O#_LOG_VAL where # in [1,5] takes one argument. The argument must be a type accepted by stream insertion operator. The evaluated value and literal text converted form of passed expresion is logged.
  • When compiling, define ENH_OPTIMISATION above the include directive as a value between 0 and 5.
  • Higher the optimisation the lower the number of logging, that is for optimisation 0, all logging O5, O4, O3, O2, O1 is active but for optimisation 5 none is active.
  • Define ENH_CLEAR_OP__ to use no logging (automatic if _DEBUG is not defined).
  • Define IGNORE_ENHANCE_DIAGNOSTICS to remove all logging calls from enhance library classes and functions.
  • Use ERROR_FLAG_LOG to log errors managed by common::error_base class template, Pass the object with which error_base methods can be called.
  • Use REPLACE for expressions to be evaluated only during debug.
  • Use REPLACE_AS for expression with different values during debug and release.
